# Bug Report Excel Structure Reference

## Overview

The Bug Reporting Agent generates a comprehensive Bug Report Excel file with two sheets:
1. **Bug Report** - Detailed list of all detected bugs
2. **Summary** - Aggregated statistics and metrics

---

## Status Workflow

```
Open → In Progress → In Review → Resolved → Closed
  ↑                                   ↓
  └─────────────────────────────────────┘
               Reopened
```

### Status Definitions

| Status | Description | Typical Duration |
|--------|-------------|-------------------|
| Open | Bug has been reported but not yet assigned or started | Until assignment |
| In Progress | Developer is actively working on fix | Development phase |
| In Review | Fix is ready and awaiting review | Review phase |
| Resolved | Fix has been implemented and passed review | Testing phase |
| Closed | Fix has been tested and verified in production | After successful deployment |
| Reopened | Bug was previously resolved but issues reappeared | After regression failure |

## Priority Levels

| Priority | Severity | Typical Fix Time | Urgency |
|----------|----------|------------------|---------|
| P1 | Critical | Within 24 hours | Critical path blocker |
| P2 | Major | Within 3-5 days | Significant functionality impact |
| P3 | Minor | Within 1-2 weeks | Non-critical issues |
| P4 | Trivial | When convenient | Cosmetic or minor issues |

---

## Severity Classification Rules

### Critical Bugs
System-breaking or blocker issues that prevent core functionality:
- Application crashes
- Authentication failures
- Security vulnerabilities
- Data loss or corruption
- Payment/checkout failures
- Server errors (500)
- Unauthorized access issues
- Blocking workflows

### Major Bugs
Functional impact or partial failures:
- Validation errors
- Broken buttons or links
- API failures
- Missing or incorrect data
- Timeout issues
- Navigation problems
- Form submission failures

### Minor Bugs
Cosmetic or non-critical issues:
- Boundary value test failures
- Minor UI inconsistencies
- Spelling/grammar issues
- Low-priority feature requests
- Minor performance issues

## Bug ID Format

Bug IDs are generated in the format: `BR-XXX`

- **BR**: Bug Report prefix
- **XXX**: Sequential number starting from 001

Examples:
- BR-001, BR-002, BR-003... (first cycle)
- BR-101, BR-102, BR-103... (second cycle, if needed)

---

## Date Format

All dates in the bug report use **YYYY-MM-DD** format (ISO 8601 date only).

Examples:
- 2026-04-04 (April 4, 2026)
- 2026-12-31 (December 31, 2026)

## Integration Notes

### With Test Execution Agent
1. Test Execution Agent runs all test cases
2. Updates test case Excel files with Status (Pass/Fail)
3. Bug Reporting Agent reads these files
4. Detects all failed test cases
5. Generates Bug_Report.xlsx with detected bugs

### With Development Workflow
1. QA generates bug report after test execution
2. Bug report is reviewed by QA Lead
3. Bugs are assigned to developers via Assigned To column
4. Developers update status using update_bug_status function
5. QA verifies fixes and updates status to Closed

### Continuous Integration
```javascript
// CI Pipeline Script
const { generateBugReport } = require('./bug-reporting-agent');

async function runBugReporting() {
  const result = await generateBugReport(
    './excel-test-cases/',
    './Bug_Report.xlsx'
  );

  if (result.bugsCount > 0) {
    // Fail CI/CD pipeline if critical bugs found
    const criticalCount = result.summary.criticalBugs;
    if (criticalCount > 0) {
      throw new Error(`CI/CD Failed: ${criticalCount} critical bugs detected`);
    }
  }
}
```

## Backup and Version Control

### Backup Strategy
1. Daily backups of Bug_Report.xlsx
2. Keep last 7 days of backups
3. Archive old reports monthly
4. Store backups in version control or cloud storage

### Version Control
```bash
# Add to Git (excluding sensitive data)
git add Bug_Report.xlsx
git commit -m "Update bug report: Added 5 new bugs"
git push

# Archive old reports
git mv Bug_Report.xlsx archive/Bug_Report_2026-04-04.xlsx
git commit -m "Archive bug report: 2026-04-04"
```

## Sample Bug Reports

### Example 1: Critical Authentication Bug
```
Bug ID: BR-001
Module: Authentication
Severity: Critical
Status: Open
Description: Users unable to login with valid credentials
Steps:
  1. Navigate to /auth/login
  2. Enter valid email "user@example.com"
  3. Enter valid password "password123"
  4. Click "Sign In" button
Expected: User redirected to /dashboard
Actual: Error "Invalid credentials" displayed
```

```
Bug ID: BR-002
Module: Marketplace
Severity: Major
Status: In Progress
Description: API timeout when fetching models list
Steps:
  1. Send GET request to /api/models
  2. Wait for response
Expected: API returns models list within 5 seconds
Actual: API timeout after 10 seconds
```

Bug ID: BR-003

## Troubleshooting

### Issue: Bug report doesn't open
**Solution**: Ensure XLSX file is not corrupted. Re-generate using the agent.

### Issue: Bug IDs are not sequential
**Solution**: Delete existing Bug_Report.xlsx and regenerate.

### Issue: Severity seems incorrect
**Solution**: Update severity manually or modify severity rules in the agent configuration.

### Issue: Cannot update bug status
**Solution**: Verify Bug_Report.xlsx exists and is not open in another application.
