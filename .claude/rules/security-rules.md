# Security Rules

- Protect private routes and sensitive APIs.
- Use JWT access tokens with refresh-token rotation.
- Hash passwords with a strong one-way algorithm.
- Validate and sanitize all external input.
- Store secrets in environment variables only.
- Restrict CORS to intended origins.
- Never expose provider keys to the client.
- Rate-limit auth and chat endpoints.
- Audit guest-session migration for account takeover risk.
