# QA Rules

- Own verification of all features before a bug is closed or a change is merged.
- Follow `communication-rules.md` for the required bug-report fields: repro steps, expected result, actual result, affected route or API, severity.
- Root rules in `global-rules.md` override QA guidance on any conflict.
- Use `workflow-qa.md` as the process companion to these rules.
- Severity taxonomy:
  - blocker: prevents all use of a core feature; must fix before release.
  - critical: data loss, auth bypass, or broken primary flow; no workaround.
  - major: significant feature broken; workaround exists.
  - minor: cosmetic or edge-case issue; low user impact.
  - trivial: polish, copy, or micro-UX issue.
- Bug lifecycle: open → triage → fix → QA verify → close. QA must verify before closing; reopen if regression.
- Every fixed bug requires a regression test before the bug is closed.
- Backend unit and E2E tests use the existing Jest scripts: `npm run test`, `npm run test:cov`, `npm run test:e2e`.
- Tests must hit a real database; do not mock the database unless the scope explicitly requires it.
- Do not write placeholder, no-op, or always-passing tests.
- Frontend has no unit test runner today; frontend QA coverage lives in Playwright.
- Use stable Playwright selectors (role, label, accessible name); avoid brittle CSS or XPath selectors.
- Do not use `sleep` or arbitrary waits in Playwright tests; use proper awaits and network-idle states.
- Retry flaky network calls only; never retry flaky assertions by adding retries to mask root cause.
- Keep Playwright traces on failure; attach them to bug reports.
- Guest-session and auth refresh flows must be covered by E2E tests given their complexity.
- Use dedicated test accounts for auth and chat tests; never use production credentials.
- Do not commit secrets, real tokens, or real user data in test fixtures.
- Rate-limit-sensitive endpoints (auth, chat) must be tested with appropriate pacing; do not hammer endpoints in tests.
- Raise blockers immediately; do not continue testing with hidden assumptions that affect another layer.
- Handoff to engineering must include all fields from `communication-rules.md`.
