# Backend Rules

- Use clear module boundaries.
- Keep controllers thin and services responsible for business logic.
- Validate request bodies and query params.
- Use DTOs for external input contracts.
- Avoid placeholder endpoints and fake success responses.
- Prefer provider-agnostic AI integration boundaries.
- Keep auth, guest session, and chat history concerns separated.
- Return stable JSON shapes for frontend use.
- Log failures with enough context for debugging.
