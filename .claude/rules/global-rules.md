# Global Rules

- Do not create new repositories.
- Do not rename `frontend/` or `backend/`.
- Respect the existing codebase before adding new patterns.
- Follow a minimal dependency policy.
- Reuse built-in framework features before adding packages.
- All buttons and routes must work.
- Avoid placeholder API calls.
- Prefer real contracts, mocks only when explicitly scoped.
- Do not change the design to a dark theme.
- Implement and preserve a modern light theme.
- Keep changes small, reviewable, and aligned with shared specs.
- Follow `.claude/rules/communication-rules.md` for all cross-scope handoffs.
