# QA Skill

- Goal:
  - Run fast, repeatable validation on routes and user flows.
- Focus areas:
  - Navbar links.
  - Chat Hub behavior.
  - Marketplace filters and CTA.
  - Login, signup, protected routes.
  - Guest-session expiry and migration.
- Execution:
  - Test public routes first.
  - Test auth routes second.
  - Re-test only affected areas after fixes.
  - Report findings with the shared communication contract.
- Output:
  - Findings first.
  - Include route, repro, expected, actual.
