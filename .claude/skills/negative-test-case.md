# Negative Test Case Skill

## Purpose
Generate invalid and failure scenarios for API testing.

---

## Responsibilities
- Missing required fields
- Invalid data types
- Unauthorized access
- Expired or missing tokens
- Invalid JSON payloads
- Wrong HTTP methods
- Forbidden actions

---

## Rules
- Ensure API returns proper error codes (400, 401, 403, 404, 500)
- Must simulate real-world failure cases
- Focus on security and validation failures
